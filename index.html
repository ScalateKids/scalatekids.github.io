Actorbase

Introducing “Actorbase”.
A NoSQL database, based on the Actor Model for concurrency and oriented to the management of heavy load of data.

Download Learn more
A completely new DSL

The DSL (Domain System Language), upon which the actorbase CLI (Command Line Interface) is based, has been developed trying to make it as close as possible to spoken English in order to make it easier to understand.

Easy to implement

Actorbase is essentially an actor system built upon a server component that exposes some API forming a REST-ful web service. To facilitate the use from inside a Scala source, a driver has been developed in order to allow the usage of all features that Actorbase provides.

Scalable

Built using the actor model for concurrency, Actorbase aims to be scalable in order to manage heavy load of data.

Akka and the actor model

We developed our product using Scala and Akka.

  • Scalable database

    As stated before, Actorbase aims to be completely scalable.

  • Actor Model

    Actorbase is build upon the actor model for concurrency proposed by Carl Hewitt in the 1973. For further infos visit the official Akka documentation

  • Clean reusable code

    Download and re-use the Actorbase open source code for any other project you like.

JVM and Actorbase portability

Actorbase is written using Scala, a general purpose functional programming lenguage.

Scala source code is intended to be compiled to Java bytecode, so that the resulting executable code runs on a Java virtual machine (JVM).

Making Actorbase running on a JVM implies to make it executable on every operating system without further ado;

We have tested our product on Windows 10, Ubuntu 14.04 LTS, OSX El Capitan and following.
